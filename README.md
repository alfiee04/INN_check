# files_check — автоматическая проверка контрагентов по открытым реестрам

## 1. Описание программы

Скрипт автоматически проходит по набору открытых государственных и
коммерческих реестров (Вестник госрегистрации, Прозрачный бизнес/ФНС,
Федресурс, картотека арбитражных дел, ФССП, Минюст и др.), делает
скриншот каждой проверенной страницы по ИНН контрагента и собирает их в
единый Word-отчёт (ГОСТ-формат) для отдела закупок/экономической
безопасности.

Реквизиты проверяемых компаний (ИНН, название, адрес и т.д.) хранятся не
в коде, а в отдельном файле `config.json` — его можно редактировать
текстом или собрать через прилагающийся HTML-конструктор
`config_builder.html` (без сервера и сторонних библиотек). В `config.json`
можно указать сразу список из нескольких компаний и проверить их за один
запуск.

---

## 2. Порядок запуска

1. Подготовить `config.json` (скопировать `config.example.json` и
   заполнить вручную, либо собрать через `config_builder.html`).

2. Установить зависимости:
   ```bash
   pip install playwright python-docx
   playwright install chromium firefox
   npm install
   ```

3. Запустить:
   ```bash
   python main.py
   ```

   Флаги:
   | Флаг | Описание |
   |---|---|
   | `--config path/to/file.json` | другой файл конфигурации |
   | `--inn 7707083893` | проверить только одну компанию из списка |
   | `--all` | проверить все компании из `companies` |
   | `--headless` | без GUI |
   | `--only vestnik fssp kad` | выполнить только указанные чекеры |

4. Результат: скриншоты в `screenshots/`, по каждой компании — отдельный
   `report_<ИНН>_<дата_время>.docx`.

## 3. Корневая структура программы

```
files_check/
├── config.py              # загружает config.json, отдаёт значения как атрибуты модуля
├── config.json             # реквизиты проверяемых компаний (не в репозитории)
├── config.example.json     # шаблон конфигурации
├── config_builder.html     # HTML-конструктор config.json
├── main.py                 # точка входа — запуск проверок
├── report.py                # сборка Word-отчёта (.docx)
├── checkers/
│   ├── __init__.py          # список всех чекеров (ALL_CHECKERS)
│   ├── base.py               # базовый класс чекера
│   ├── vestnik_gosreg.py, fedresurs.py, fssp.py, kad_arbitr.py, ...
│   └── address_search.py     # поиск адреса в Яндексе через отдельный Firefox
├── screenshots/              # скриншоты проверок
└── package.json / package-lock.json  # Node-зависимости (Playwright, @aluvia/sdk)
```

## 4. Неточности и нюансы, которые нужно доработать

- **vestnik-gosreg.ru** (`https://www.vestnik-gosreg.ru/search/`) — стоит
  Яндекс-капча, требует ручного ввода слов с картинки. Автообход не
  реализован и не предусмотрен — чекер ждёт, пока сотрудник пройдёт
  капчу в открытом окне браузера.

- **fedresurs.ru** — сайт периодически не загружается. Иногда помогает
  открытие через мобильные прокси, но в целом в тестовом режиме поиск по
  ИНН не проходит.

- **kad.arbitr.ru** — в тестовом режиме кнопка «Найти» не нажимается ни
  кодом (Playwright), ни вручную — требуется отдельное разбирательство,
  возможно смена подхода к взаимодействию с формой поиска.

- **fssp.gov.ru** — стоит сложная капча, автоматический проход
  невозможен, нужен аналогичный ручной режим ожидания, как для
  vestnik-gosreg.ru.

- **gosuslugi.ru** (`https://www.gosuslugi.ru/621102/1`) — требует
  авторизации через Госуслуги, из скрипта недоступен. Нужно заменить
  источник на `https://хочу-в-суд.рф/` (`https://xn----7sbbags9a3agcgbedhnu.xn--p1ai/`)
  и переписать соответствующий чекер под новую вёрстку.
